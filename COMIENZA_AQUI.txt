⚡ INSTALACIÓN RÁPIDA (5 MINUTOS)
════════════════════════════════════════════════════════════════════════════════

Paso 1: Ejecutar Scripts SQL en SQL Server
─────────────────────────────────────────
1. Abre SQL Server Management Studio
2. Abre: Backend/db_scripts/modulos_structure.sql
3. Presiona F5 (ejecutar)
4. Verifica que dice "Tablas de módulos creadas exitosamente"

5. Abre: Backend/db_scripts/modulos_data_initial.sql
6. Presiona F5 (ejecutar)
7. Verifica que dice "Datos iniciales insertados exitosamente"


Paso 2: Reiniciar Backend Django
─────────────────────────────────
1. Abre terminal en: Backend/
2. Ejecuta: python manage.py runserver
3. Verifica: "Starting development server at http://127.0.0.1:8000/"


Paso 3: Reiniciar Frontend React
────────────────────────────────
1. Abre terminal en: Frontend/
2. Ejecuta: npm run dev
3. Verifica: "VITE v... ready in ... ms"


Paso 4: Probar en Navegador
──────────────────────────
1. Abre http://localhost:5173
2. Inicia sesión como: admin / [tu contraseña]
3. En el sidebar, al final: "⚙️ Administración de Módulos"
4. Haz clic
5. Selecciona un usuario
6. Arrastra un módulo de izquierda a derecha
7. ¡Debería moverse automáticamente!

✅ ¡LISTO! Sistema funcionando

🎯 CASOS DE USO
════════════════════════════════════════════════════════════════════════════════

1. ADMIN ASIGNA MÓDULOS A USUARIO:
   • Haz clic en "Administración de Módulos"
   • Selecciona usuario "juan"
   • Panel izquierdo: "Usuarios", "Asistencias", "Exámenes"
   • Panel derecho: (vacío)
   • Arrastra "Usuarios" a derecha
   • ¡Ahora juan tiene acceso a Usuarios!
   • Los cambios se guardan automáticamente en BD

2. ADMIN DESASIGNA MÓDULO:
   • Panel derecho: "Usuarios"
   • Arrastra a izquierda
   • ¡Se desasignó!
   • En BD: ACTIVO = 0

3. BUSCAR MÓDULOS:
   • Escribe en caja de búsqueda
   • Se filtra en tiempo real
   • Funciona en ambos paneles

❓ PREGUNTAS FRECUENTES
════════════════════════════════════════════════════════════════════════════════

P: ¿Dónde aparece "Administración de Módulos" en el menú?
R: Solo para admins, al final del sidebar con ícono ⚙️

P: ¿Qué usuarios aparecen en el dropdown?
R: Los de la tabla USUARIO en BD (por ahora, rellenar manual o crear endpoint)

P: ¿Se guardan los cambios automáticamente?
R: Sí, cuando arrastras o haces clic en botón, se guarda inmediatamente

P: ¿Qué permisos puedo asignar?
R: read, write, delete, admin (o combinarlos en JSON)

P: ¿Puedo agregar un nuevo módulo?
R: Sí, con INSERT en tabla MODULO (sin tocar código)

P: ¿Es escalable?
R: 100% escalable. Agrega módulos, roles, permisos sin cambiar código

P: ¿Se puede usar con SQL Server?
R: Sí, completamente. Está diseñado para SQL Server con BD en MAYÚSCULAS

P: ¿Secretarios y usuarios ven "Administración de Módulos"?
R: No, solo admins. Está protegido.


════════════════════════════════════════════════════════════════════════════════
🆘 SI ALGO NO FUNCIONA
════════════════════════════════════════════════════════════════════════════════

1. Módulos no aparecen:
   └─ Ejecuta: Backend/db_scripts/modulos_verify.sql
   └─ Debe mostrar ✓ para todas las verificaciones

2. Error "404" en API:
   └─ Reinicia Django: python manage.py runserver
   └─ Limpia navegador: CTRL+SHIFT+R

3. Dropdown de usuarios vacío:
   └─ Revisa: SELECT * FROM USUARIO WHERE ESTADO = 'Activo'
   └─ Inserta usuarios si no hay

4. Drag-and-drop no funciona:
   └─ Abre console (F12)
   └─ Revisa si hay errores de JavaScript
   └─ Prueba en navegador diferente

5. No aparece el menú de admin:
   └─ ¿Iniciaste sesión como admin?
   └─ Revisa BD: IDTIPOUSUARIO = 3 (admin)
   └─ Limpia localStorage: Abre DevTools > Application > Clear all
